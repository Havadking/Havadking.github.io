Java学习笔记

1 Java SE 基础

1.1 面向对象基础

1.1.1 什么是类

  • 类是对现实生活中一类具有相同属性和行为 的事物的抽象。
    • 类的属性由成员变量体现。
    • 类的行为由成员方法体现（去掉static关键字）。

1.1.2 什么是对象

  • 看得到摸得着的真实存在的实体。
  • 类是对象的抽象，对象是类的实体。
  • 对象具有属性和行为。

1.1.3 Java内存分配

  • 栈：所有局部变量都会在栈内存中创建。

    • 方法执行都会加载到栈里。
    • 方法执行完会从栈内存中消失。

  • **堆：**所有对象及其对应的实例变量和数组会放到堆内存中。

    • new出来的东西

    • new出来的有一个地址值，用来被空闲时回收

    • 实例变量具有默认初识值。

  • 单个对象

  • 多个对象

  • 多个引用指向相同

1.1.4 关键字和构造方法

  • 成员变量和局部变量

  • JavaBean

    • java的一个类，其对象可以在程序中封装数据。
    • 满足：变量private修饰；对应set/get方法；提供一个无参构造方法；

1.2 API 基础

1.2.1 Scanner

  • next和nextline的区别:
    • next():遇到空格不在录入。
    • nextline():可以将数据完整接收过来。遇到回车、换行符结束录入。
  • nextline和nextint同时使用会出现问题
    • nextint在前面时，输入了回车导致nextline无法接受到输入值。

1.2.2 String

  • Java中所有的字符串都是String的对象。
  • 字符串是一个常量，其值是不可更改的。
  • String是一个特殊类，打印其对象名时不会出现内存地址，而是其记录的真实内容。
  • ==号比较
    • 基本数据类型：比较具体的值。
    • 引用数据类型：比较地址值。
  • 通过双引号创建字符串对象时，字符串相同时，无论代码中创建几遍，JVM都只创建一个，并在字符串常量池中维护。
    • 字符串常量池：双引号创建字符串时，系统会检查其是否存在在字符串常量池中。
  • 通过new创建时，每次都会申请一个新的内存地址。

1.2.3 StringBuilder

  • 可变的字符串类，可以将其看作为一个容器。
  • 作用：提高字符串的操作效率。

  • 分包思想（不同功能的类文件，放在不同的包下）:
    • 本质是文件夹
    • 命名规则：字母都是小写的

  • static关键字特点
    • 被static修饰的成员会被该类的所有对象共享
    • 被static修饰的成员会随着类的加载而加载，优先于对象存在
    • 非静态需要创建对象之后，才能进行使用
    • 多了一种调用方式，可以通过**类名.**进行调用

2.1.3 继承

  • 继承的好处
    • 调高代码的复用性
    • 提高代码的维护性
    • 类与类之间产生关系，提升了多态
  • 继承的坏处
    • 降低代码的灵活性
    • 增强了代码的耦合性
  • this关键字和super关键字
    • 就近原则： 子类->成员->父类
  • 方法重写
    • 子类和父类出现一模一样的方法生成
    • 子类需要父类功能，又有自己特有的内容：通过super先调用
    • 父类中私有的方法不能被重写
    • 静态方法必须静态重写，非静态也相同
      • 静态方法不能被重写，其本质是父类的静态方法被覆盖掉
    • 重写时，子类权限必须大于等于父类
  • 权限修饰符
  • 每一个构造方法中，都默认存在了一个super()空参构造方法
  • 抽象类的特点：
    • 不能创建对象
    • 抽象类中有构造方法
    • 子类必须重写所有抽象方法
    • 子类可以将自己也变成一个抽象类
    • 抽象类中可以没有抽象方法，但有抽象方法的类肯定是抽象类
  • 模板设计模式
    • 把抽象类整体看作一个模板，让继承类去重写抽象方法实现需求
  • final关键字
    • 被其修饰的方法不能被重写了
    • 被其修饰的类不能再被继承了

​ 常量的命名规范：

​ 1.一个单词所有字母大写

​ 2.多个单词所有字母大写，中间用_隔开

  • 代码块
    • java中{ }的代码称为代码块
    • 静态代码块
      • 类中方法外
      • static关键字修饰，随着类的加载而加载，并且只执行一次
      • 在类加载时做一些初始化操作
    • 局部代码块
      • 方法当中定义
      • 限定变量的生命周期，及早释放，提高内存的利用率
    • 构造代码块
      • 类中方法外定义
      • 构造方法执行时执行，且在构造方法前执行
      • 将重复的构造方法中的内容添加到构造代码块中

2.1.4 接口

  • 一个类中，所有的方法，都是抽象方法，可以将其定义为接口
  • 接口是一种引用数据类型
  • 接口存在的两个重要意义：
    • 规则的定义
    • 程序的拓展性
  • 定义interface 实现implement
  • 接口和类之间的关系可以单实现也可以多实现
  • 接口中的成员变量只能是常量，其默认被static,final,public关键字修饰
  • 接口中无构造方法
  • 接口中的成员方法默认有public,abstract修饰
  • JDK8
    • 可以写非抽象方法，用default关键字
    • 可以写static静态方法
  • JDK9中允许方法使用private修饰
  • 直接父类和接口出现相同的方法，优先使用父类的方法

2.1.5 多态与内部类

  • 同一个对象，在不同时刻表现的不同形态
  • 前提条件
    1. 继承/实现关系
    2. 方法重写
    3. 父类引用指向子类对象
  • 多态的特点
    • 成员变量：编译看左边（父类），运行看左边（父类）
    • 成员方法：编译看左边（父类），运行看右边（子类）
  • 好处：提高了代码的扩展性
    • 使用父类作为参数，可以接收任意的子类对象
  • 弊端：不能调用子类特有的方法
  • 多态中的转型：
    • 向上转型：父类引用指向子类对象
    • 向下转型：子类引用指向父类对象

2.1.6 Lambda

  • 方便理解：Lambda表达式，对匿名内部类进行类优化

  • 函数式编程思想：更多关注做什么

    • 尽量忽略面向对象的复杂语法

  • Lambda表达式的三要素：形式参数，箭头，代码块

  • Lambda表达式的格式：

    (形式参数) -> (代码块)

  • Lambda表达式前提：

    1. 有一个接口
    2. 接口中有且仅有一个抽象方法

  • 匿名内部类和Lambda的区别

    • 类型不同
      • 匿名内部类：可以接口，抽象类，或具体类
      • Lambda：只能接口
    • 限制不同
      • 接口中仅有一个抽象方法：Lambda或匿名内部类
      • 接口中有多个抽象方法：只能用匿名内部类
    • 实现原理不同
      • 匿名内部类：编译后，产生一个单独.class文件
      • Lambda:编译后，不会生成.class文件。其在运行过程中动态生成，不会保存在电脑中。

2.2.9 异常

  • 用来描述代码中出现的问题，导致JVM非正常停止。

  • Throwable

    • Error:严重问题，无法通过代码处理。比如：内存溢出
    • Exception:异常类，程序本身可以处理的问题
      1. RuntimeException及其子类:运行时异常（空指针，数组索引越界）
      2. 除RuntimeException之外的所有异常:编译时异常，程序不能通过编译。(日期格式化异常)
    • 成员方法
      1. getMessage:返回详细消息
      2. toString:返回可抛出的简短描述
      3. printStackTrace:把异常信息输出到控制台**(程序还会继续执行下去)**

  • JVM虚拟机处理异常

    1. 将异常信息以红色字体输出在控制台
    2. 在出现异常出停止程序运行

  • throws 异常类名;

    • 写在方法定义出，声明一个异常。
    • 如果声明的异常是一个运行时的异常，那么可以省略不写
    • 如果声明的异常是一个编译时的异常，必须在方法后声明

  • throw

    • 创建异常的对象
    • throw new 异常; 创建后下面的代码不在执行

  • throws和throw的区别

  • 异常的处理方法 try…catch…

    try{
    //可能异常的代码
}catch(异常类){
    出现异常进行的操作
}
    • 好处：可以让程序继续执行下去
    • 多个异常可以用多个catch来实现

  • 自定义异常

    • 目的：让异常的信息更加的清晰可读
      1. 定义异常类(类名见名知意)
      2. 写继承关系
      3. 写空参构造
      4. 写带参构造

        2. LinkedList
      2. Set不可重复（接口）

2.3.2 List集合

  • 有序集合，有序指的是存取顺序

  • 特点：

    1. 有序：存储和取出顺序相同
    2. 有索引
    3. 可重复

  • ArrayList

    • 底层数据结构为数组，查询快，增删慢

    • 空参构造时在底层创建了一个长度为0的数组

    • 默认长度为10，默认名字elementDate，默认变量size

  • LinkedList

    • 底层数据结构为链表，查询慢，增删快
    • 双向链

2.3.3 泛型

  • 使用的地方

    • 方法声明上

  • 定义格式

    • <类型1,类型2>:两种格式

  • 泛型方法

    • public static void add(T t1, T t2);

  • 泛型接口

    • 实现类也不给泛型
    • 实现类给出具体的数据类型

  • 通配符

    • 类型通配符：<?>
    • 类型通配符上限：<?extends类型>：表示类型及其子类
    • 类型通配符下限：<?super类型>:表示类型及其父类

2.3.4 Set集合

  • 可以去重复
  • 存取顺序不一致
  • 没有带索引的方法，不能使用普通for遍历
  • TreeSet
    • 可以对内部的元素按照规则进行排序
    • 要使用TreeSet必须指定排序规则
    • 排序规则：
      1. 自然排序
        1. 使用空参构造TreeSet集合
        2. 自定义类中实现Comperable接口
        3. 重写compareTo方法
          1. comperTo返回值为负数，表示较小值，放在左边
          2. 0，重复了，不存
          3. 正数，较大值，放在右边
      2. 比较器排序
        1. 使用带参构造方法
        2. 让集合构造方法接收Comparator的实现类对象，重写compare(T o1, T o2)方法
        3. 必须按照主要条件和次要条件来写
    • 数据结构-树
      • 二叉查找树
        1. 每一个节点只有两个字节点
        2. 左子节点小于自己
        3. 右子节点大于自己
      • 平衡二叉树
        • 是一个二叉查找树
        • 特点
          1. 左右两子树的高度差不超过1
          2. 任意节点的左右两个子树都是平衡二叉树
        • 左旋
        • 右旋
      • 红黑树
        • 平衡二叉B树
        • 每个节点都可以是红或者黑
        • 是一个二叉查找树，但不是高度平衡的，通过自己的红黑规则来实现平衡
        • 红黑规则：
          1. 每一个节点必须是红色或黑色
          2. 根节点是黑色的
          3. 如果一个节点没有字节点或父节点，则该节点相应的指针属性值为Nil，这些Nil视为叶节点，且都是黑色的
          4. 如果一个节点是红色的，他的字节点必须是黑色的
          5. 对每一个节点，该节点到其所有后代叶节点的简单路径上，均包含相同数目的黑色节点
