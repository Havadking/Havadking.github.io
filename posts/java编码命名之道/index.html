Java编码命名之道

常见的命名规则

一、大驼峰命名法

  • 类名使用
  • 例子：ServiceDiscovery, ServiceInstance, LruCacheFactory

二、小驼峰命名法

  • 方法名，参数名，成员变量，局部变量
  • 例子：getUserInfo(), String nameFormat

三、蛇形命名法

  • 测试方法类，常量，枚举名称
  • 例子：CLIENT_CONNECT_SERVER__FAILURE

四、串式命名法

  • 项目文件可以使用
  • 例子：dubbo-all, dubbo-bom

常见命名规范

Java语言基本命名规范

  1. 类名使用大驼峰命名法（UpperCamelCase）。方法名、参数名、成员变量、局部变量使用小驼峰命名法（lowerCamelCase）。
  2. 测试方法名、常量、枚举使用蛇形命名法。测试方法名要求全部小写，常量以及枚举需要全部大写。
  3. 项目文件夹名称使用串式命名法。
  4. 包名统一使用小写，尽量使用单个名词作为包名，各个单词之间通过.分隔符连接。
  5. 抽象类命名使用Abstract开头。
  6. 异常类命名使用Exception开头。
  7. 测试类命名以它要测试的类的名称开始，以Test结尾。

命名易读性规范

  1. 为了更加易读和易懂，尽量不要使用缩写/简写单词，除非这些单词被公认为可以被这样缩写/简写。
  2. 命名不像函数一样追求短，可读性优先于简短的名字，虽然可读性的名字会比较长一点。
  3. 避免无意义的命名，每个名字都要能表明意思。比如避免int count,而要int userCount。
  4. 避免命名过长（50个字符以内最好）。
  5. 不要使用拼音，更不要使用中文。
